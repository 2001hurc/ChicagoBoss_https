May 22, 2010 – Chicago Boss 0.3.2 is out (download) and has major, major performance improvements, as well prettier error messages, pagination on the /admin interface, and boolean operator support in the template language. By the way, if you haven't seen it already, be sure to check out the beginner's guide, “An Evening With Chicago Boss”.

How is Chicago Boss different from other Erlang MVC frameworks?

Chicago Boss features:

  • Erlang's most advanced ORM. Boss's Object-Relational Mapping, called BossRecord, uses some compiler magic to add getters, setters, savers, and a host of other useful methods to your parameterized modules, which persist in a Tokyo Tyrant database. Check out the BossRecord API if you are in a state of disbelief.

  • Erlang's best templating engine. Django Template Language gets compiled right down to BEAM (Erlang byte code) with ErlyDTL. In development mode, templates are automatically recompiled every time you reload a page.

  • Pattern-matching controllers. Unlike other frameworks where you have to specify URL-matching patterns in a separate file, Chicago Boss uses Erlang's built-in function-matching to map a URL to an action. This design makes for the cleanest controllers on the market. The Controller API has details.

  • Bonus features. Chicago Boss automatically generates documentation and an admin interface for each of your models. You don't even have to type "make".

Why should I switch to Erlang from Python/Ruby/Perl/PHP/FORTRAN?

Because Erlang is the future of web programming. No other platform uses a purely asynchronous I/O strategy or has the same scalability and availability. Chicago Boss lets you take advantage of it all while writing clean, simple, elegant web apps. If you're new to Erlang, be sure to Learn You Some Erlang For Great Good!.

I don't want to learn Boss only to find out it's missing Feature X. What features does Chicago Boss lack?

Great question! Chicago Boss does NOT have:

  • Built-in tools for generating RSS
  • Built-in tools for sending email
  • Built-in deployment or monitoring tools
  • Internationalization support
  • Support for partitioned databases
  • Support for databases other than Tokyo Tyrant

We're anxious for these features, too, and invite you to submit your ideas or patches to the Chicago Boss mailing list.

How mature is Chicago Boss?

Chicago Boss is still in beta, but it's getting better every day. The API is small, but performance is great (as of version 0.3.2, at any rate). There might be minor changes to the API before the 1.0 release, but Boss is being used on low-traffic production sites.
