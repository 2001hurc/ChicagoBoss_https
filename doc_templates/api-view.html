Chicago Boss liked Django's templating language so much, he decided to steal it. Template files go in your project's View folder, and will have access to the variables you pass from your Controller.

Note: if you use the the "extends" tag, the file path should be relative to your project's "View" directory.

Chicago Boss has support for 99% of the Django Template Language (everything except the "load" tag), so existing Django templates should work out of the box. If your template can be compiled using ErlyDTL, Boss compiles the template down to Erlang BEAM code to give you the fastest templates on the planet. If your templates need features not supported by ErlyDTL, they'll be rendered with Etcher instead.

To make sure your templates are compiled to BEAM code, you need to restrict yourself to the following filters. Hopefully this list will grow over time.
