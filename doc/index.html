How is Chicago Boss different from Rails/Django/Mason/CakePHP/My Favorite MVC Framework?

All right, enough with the Boss-speak, I'll give it to you straight: Chicago Boss was designed by a poor grad student to handle lots of traffic on minimal hardware. Like Nginx or Memcached, Chicago Boss uses an event-driven I/O strategy to handle hundreds or thousands of simultaneous requests, and thus it solves the classic c10k problem. Chicago Boss is built with Erlang, the same platform used by banks and telecoms to achieve 99.9999999% reliability.

How mature is Chicago Boss?

Chicago Boss is still in beta. There might be minor changes to the API before the 1.0 release, but it's being used on (admittedly low-traffic) production sites.

Wait, will I need to learn new things to use Chicago Boss? Please don't make me learn new things.

Yes and no. Chicago Boss has ripped off several Rails programming conventions (like ActiveRecord) as well as the complete Django Template Language in order to make your life as easy as possible. But you need to know at least a little Erlang to do anything useful with Boss.

Does Chicago Boss support MySQL/Postgres/Ingres/CouchDB/Microsoft Excel?

No/No/No/No/No! Boss refuses to talk to anyone except Tokyo Tyrant. Check out the section on Tyrant in the "Evening With Chicago Boss" for installation instructions.
